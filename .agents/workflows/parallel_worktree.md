---
description: 새로운 기능을 병렬로 개발할 때, 기존 작업 내역 충돌 없이 빠르고 독립적으로 세팅하는 워크트리 자동화 규칙
---

이 워크플로우는 기존 작업(ex. 현재 브랜치의 수정 사항)에 영향을 주지 않고 완전히 독립된 환경에서 새로운 브랜치 작업을 해야 할 때 자동으로 실행됩니다.

// turbo-all
1. `git branch`와 `git status`를 실행하여 현재 환경 상태를 파악합니다.
2. 새롭게 작업할 '브랜치 이름'이 명시되어 있는지 확인합니다. (없다면 사용자에게 물어봅니다.)
3. 상위 경로(`../`)에 `FrameUI-[브랜치이름]` 형태로 독립된 디렉토리를 생성할 계획을 세웁니다.
4. `git worktree add -b [브랜치이름] ../FrameUI-[브랜치이름]` (만약 이미 존재하는 브랜치라면 `-b` 생략) 명령어를 실행하여 새로운 워크트리 공간을 생성합니다.
5. 그 후 생성된 새로운 폴더로 진입하여 필요한 패키지(`pnpm install`)가 있다면 설치해 개발 준비를 마칩니다.
6. 세팅이 끝나면 새로운 워크트리 공간에서 바로 작업을 시작하겠습니다.
